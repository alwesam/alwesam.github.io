Lesson of the Day: Cookie Flags

May 2nd, 2018 12:35 pm

HttpOnly

When added to the http request header, HttpOnly flag tells the browser to only share its data with the server (through http requests). This prevents potential tampering by client-side scripts.

Secure

The secure flag forces the browser to only send its cookies through secure http line (https). It becomes redundant when https connection is already established but could be useful when the security of the client-server connection is degraded for whatever reason, in which case no cookies will be shared.

Domain

The domain flag ensures the cookies are sent to specific hosts. It defaults to the host portion of the current document (but not including subdomains).
